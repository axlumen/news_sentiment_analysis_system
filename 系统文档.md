# 新闻情感分析系统 - 系统文档

## 1. 系统概述

### 1.1 系统简介
基于Django框架的Web应用，集成新闻爬虫、数据预处理、深度学习情感预测和统计分析功能，实现从数据采集到情感分析的完整自动化流程。

### 1.2 系统架构
- **Web展示层**：Django用户界面
- **业务逻辑层**：数据处理、情感预测、统计分析
- **数据存储层**：MySQL数据库
- **AI模型层**：Bi-LSTM+Attention情感分类模型

### 1.3 主要功能
- 新闻数据自动爬取（多板块支持）
- 数据清洗与预处理
- 中文文本情感分析（三分类：正面、中性、负面）
- 情感数据统计与可视化
- 用户管理与认证系统

## 2. 系统模块详解

### 2.1 Web应用层

#### 2.1.1 应用配置 (news_sentiment_analysis/)
- **settings.py**: Django系统配置（数据库、应用注册、中间件、静态文件）
- **urls.py**: 系统路由（/admin/、/、/user/）

#### 2.1.2 主应用模块 (app/)
- **models.py**: 数据模型（NewsRaw、NewsSentimentAnalysis、各类统计表）
- **views.py**: 视图函数（index、news_list、api_run_pipeline）
- **urls.py**: 应用路由（/、/news-list/、/api/run-pipeline/）
- **admin.py**: 管理后台注册

#### 2.1.3 用户模块 (user/)
- **models.py**: 自定义用户模型（包含头像字段）
- **views.py**: 用户视图（登录、登出、注册、个人中心、头像上传）
- **urls.py**: 用户路由

### 2.2 数据处理层 (data/)

#### 2.2.1 新闻爬虫 (news_spider.py)
- **功能**: 新浪新闻多板块数据爬取
- **支持板块**: 综合新闻、财经、科技、体育、娱乐、教育、健康、时尚
- **关键类**: NewsMySQL（数据库操作）、TextCleaner（文本清洗）、SinaSpider（爬虫主类）
- **配置**: MAX_TOTAL_NEWS=200（总上限）、MAX_CATEGORY_NEWS=10（单分类上限）

#### 2.2.2 数据预处理 (data_preprocess.py)
- **功能**: 数据清洗、标准化、去重
- **流程**: 读取原始数据 → 数据校验 → 重复值检测 → 文本清洗 → 标准化 → 保存处理结果

#### 2.2.3 情感预测 (predict_sentiment.py)
- **功能**: 使用Bi-LSTM+Attention模型进行情感分析
- **标签映射**: 0=负面、1=正面、2=中性
- **主要函数**: process_existing_records（处理未分析记录）、predict_batch（批量预测）、predict_single（单条预测）

#### 2.2.4 统计分析 (news_analysis.py)
- **功能**: 生成情感统计报表
- **维度**: 整体、来源、分类、日期、得分分布、关键词提取

#### 2.2.5 主流程控制 (main.py)
- **功能**: 协调各模块执行完整数据流程
- **顺序**: 新闻爬虫 → 数据预处理 → 情感预测 → 统计分析
- **选项**: --skip-spider、--skip-preprocess、--skip-predict、--skip-analysis

### 2.3 AI模型层 (model/)

#### 2.3.1 模型结构
- **BiLSTMAttention**: Bi-LSTM + Attention模型
  - 词嵌入层 → 双向LSTM层 → Attention层 → 全连接层

#### 2.3.2 训练与预测
- **train.py**: 模型训练脚本（优化内存使用）
- **predict.py**: 模型预测脚本（修复三分类逻辑）
- **checkpoints/best_model.pth**: 最佳模型权重
- **data/processed/vocab.pkl**: 词表

#### 2.3.3 数据准备
- **init_data.py**: 数据初始化（支持读取neg.csv、neutral.csv、pos.csv）
- **preprocess.py**: 词表构建与文本预处理
- **utils/dataset.py**: 数据集封装
- **utils/plot_curve.py**: 训练曲线绘制

### 2.4 前端展示层 (templates/)

#### 2.4.1 页面结构
- **base.html**: 基础模板
- **index.html**: 首页（数据爬取按钮、统计展示）
- **components/**: 头部、侧边栏组件
- **analysis/**: 数据分析仪表板、新闻列表
- **auth/**: 登录、注册、个人中心页面

#### 2.4.2 静态资源 (static/)
- **css/**: 样式文件
- **js/**: JavaScript文件（Chart.min.js、charts-home.js、front.js）
- **fonts/**: 字体文件
- **vendor/**: 第三方库

## 3. 数据库设计

### 3.1 数据表结构

#### 3.1.1 原始数据表 (app_newsraw)
- **主要字段**: id, title, content, source, url, category, publish_date, read_count, comment_count, like_count, create_time

#### 3.1.2 情感分析表 (app_newssentimentanalysis)
- **主要字段**: id, title_clean, content_clean, title_standard, content_standard, source, category, publish_date, url, read_count, comment_count, like_count, sentiment, sentiment_score, title_len, content_len, content_token_len, create_time

#### 3.1.3 统计表
- overall_sentiment（整体情感）、source_sentiment（来源情感）、category_sentiment（分类情感）、date_sentiment（日期情感）、score_distribution（得分分布）、sentiment_keywords（情感关键词）

#### 3.1.4 用户表 (user_customuser)
- **主要字段**: id, password, last_login, is_superuser, username, first_name, last_name, email, is_staff, is_active, date_joined, avatar, nickname

## 5. 系统流程

### 5.1 数据全流程
1. 新闻爬虫 → 保存到 app_newsraw 表
2. 数据预处理 → 从 app_newsraw 读取，清洗后保存到 app_newssentimentanalysis
3. 情感预测 → 处理 app_newssentimentanalysis 中未分析的记录
4. 统计分析 → 生成情感统计报表

### 5.2 用户流程
1. 用户注册/登录
2. 点击首页"数据爬取"按钮
3. 系统自动执行完整数据流程
4. 查看首页统计数据和新闻列表

## 7. 部署说明

### 7.1 环境要求
- Python 3.8+
- MySQL 5.7+
- 虚拟环境

### 7.2 安装步骤
1. 克隆项目
2. 创建虚拟环境：`python -m venv .venv`
3. 激活虚拟环境
4. 安装依赖：`pip install -r requirements.txt`
5. 配置数据库（config.py）
6. 执行迁移：`python manage.py migrate`
7. 创建超级用户：`python manage.py createsuperuser`
8. 启动服务：`python manage.py runserver`

### 7.3 数据爬取
- 首页点击"数据爬取"按钮
- 或命令行执行：`python data/main.py`
