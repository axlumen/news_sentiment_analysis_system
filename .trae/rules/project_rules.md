# 新闻情感分析项目规范

## 1. 项目结构

```
model/
├── main.py                 # 主入口，命令行接口
├── bilstm_attention.py     # Bi-LSTM + Attention 模型定义
├── train.py                # 模型训练逻辑
├── predict.py              # 模型测试与可视化
├── data/                   # 数据处理模块
│   ├── __init__.py
│   ├── init_data.py        # 数据预处理与划分
│   └── preprocess.py       # 词表构建与文本预处理
└── utils/                  # 工具模块
    ├── __init__.py
    ├── dataset.py          # 数据集封装
    └── plot_curve.py       # 训练曲线绘制
```

## 2. 技术栈

- **编程语言**：Python 3.10+
- **深度学习框架**：PyTorch
- **自然语言处理**：Jieba 分词
- **数据处理**：Pandas
- **可视化**：Matplotlib
- **其他依赖**：OpenCC（繁简体转换）

## 3. 代码风格

### 3.1 命名规范
- **函数和变量**：使用蛇形命名法（snake_case）
- **类**：使用帕斯卡命名法（PascalCase）
- **常量**：使用全大写字母和下划线（UPPER_CASE）
- **模块**：使用小写字母和下划线

### 3.2 代码格式
- 使用 4 个空格进行缩进
- 每行代码长度不超过 100 字符
- 函数和类之间空 2 行
- 函数内部逻辑块之间空 1 行
- 导入模块按标准库、第三方库、本地模块顺序排列

### 3.3 文档规范
- 每个模块、函数和类都必须有详细的文档字符串
- 文档字符串使用中文
- 函数文档应包含：功能描述、参数说明、返回值说明
- 复杂逻辑应添加注释

## 4. 数据处理规范

### 4.1 数据格式
- 原始数据：CSV 格式，包含文本和情感标签
- 预处理后数据：按 8:1:1 划分为训练集、验证集和测试集
- 情感标签：0=消极，1=中性，2=积极

### 4.2 文本预处理
- 去除 HTML 标签
- 繁体转简体
- 空白字符规范化
- 文本长度过滤（最小长度 5）
- 去重处理

### 4.3 词表构建
- 基于训练集构建词表
- 使用 Jieba 分词
- 词表大小根据数据自动确定
- 保存为 pickle 格式

## 5. 模型规范

### 5.1 模型架构
- Bi-LSTM + Attention 模型
- 输入层：词嵌入
- 隐藏层：双向 LSTM
- 注意力层：基于隐藏状态的注意力机制
- 输出层：3 分类（消极、中性、积极）

### 5.2 训练参数
- 优化器：Adam
- 学习率：1e-3
- 批量大小：64
-  epochs：100
- 早停策略：patience=10

### 5.3 评估指标
- 准确率（Accuracy）
- 精确率（Precision）
- 召回率（Recall）
- F1 分数
- 混淆矩阵

## 6. 命令行接口

### 6.1 基本命令
- `python main.py init_data`：数据预处理
- `python main.py preprocess`：构建词表
- `python main.py train`：训练模型
- `python main.py predict`：测试模型
- `python main.py plot_curve`：绘制训练曲线

### 6.2 完整流程
- 直接运行 `python main.py` 执行完整流程
- 顺序：数据预处理 → 构建词表 → 训练模型 → 测试模型 → 绘制曲线
